# Техническое задание: StudentHub / CollegeOS

## 1. Назначение проекта

**StudentHub** — персональное веб-приложение для управления учебной жизнью, которое объединяет:

* недельное планирование из Google Sheets;
* расписание и временные слоты из Google Calendar;
* данные из личного кабинета колледжа: уроки, долги, оценки, тесты, домашние задания;
* личные задачи, цели, заметки и напоминания.

Главная цель приложения — дать пользователю единый центр управления учебой: что сегодня делать, какие есть дедлайны, где есть долги, какие оценки, когда свободное время и что лучше запланировать.

---

## 2. Цели продукта

### Основные цели

1. Объединить учебное расписание, задачи, долги, тесты и оценки в одном интерфейсе.
2. Автоматически подтягивать данные из Google Calendar и Google Sheets.
3. Позволить вручную или автоматически добавлять данные из личного кабинета колледжа.
4. Показывать пользователю понятный план на день и неделю.
5. Напоминать о важных задачах, тестах, дедлайнах и долгах.
6. Помогать распределять задачи по свободным слотам в календаре.

### Дополнительные цели

1. Отслеживать прогресс по предметам.
2. Анализировать нагрузку на неделю.
3. Показывать статистику продуктивности.
4. В будущем добавить AI-помощника для планирования.
5. В будущем добавить браузерное расширение для импорта данных из личного кабинета.

---

## 3. Целевая аудитория

Основной пользователь — студент колледжа, который хочет:

* видеть все учебные дела в одном месте;
* не забывать про долги, тесты и домашние задания;
* заранее планировать неделю;
* контролировать оценки;
* лучше понимать, когда есть свободное время;
* автоматизировать рутинное планирование.

На первом этапе приложение рассчитано на одного пользователя — владельца проекта.

## 4. Основные сущности системы

### 4.1. Пользователь

Поля:

* часовой пояс;
* настройки уведомлений;

Поля:

* id;
* название;

* цвет;
* описание;
* средняя оценка;
* количество долгов;
* активен / архивирован.

* id;

* дата и время начала;

### 4.4. Задача

Поля:

* id;
* title;

* subject_id;

* приоритет: low / medium / high / urgent;

* предполагаемая длительность;
* дата создания;
* дата выполнения;

### 4.5. Долг

Поля:

* id;
* subject_id;
* title;
* description;
* дата появления;
* дедлайн;
* статус: open / planned / submitted / closed;

* ссылка на задание;
* комментарий пользователя.

### 4.6. Оценка

Поля:

* id;
* subject_id;
* value;
* max_value;
* тип: тест / самостоятельная / домашняя / проект / устный ответ / другое;
* дата получения;
* вес оценки;
* комментарий;
* источник.

### 4.7. Тест / контрольная

Поля:

* id;
* subject_id;
* title;
* дата и время;
* дедлайн;
* статус: upcoming / completed / missed;
* результат;
* ссылка на тест;
* уровень важности;
* нужно ли готовиться;
* план подготовки.

### 4.8. Недельный план

Поля:

* id;
* week_start_date;
* цели недели;
* задачи недели;
* заметки;
* статус выполнения;
* источник: google_sheets / manual.

### 4.9. Напоминание

Поля:

* id;
* title;
* description;
* remind_at;

### 4.10. Заметка

Поля:

* дата создания;

## 5. Функциональные требования

## 5.1. Авторизация

### Обязательные функции

* Пользователь может войти через Google OAuth.
* После входа приложение получает доступ к Google Calendar.
* Отдельно пользователь может подключить Google Sheets.
* Пользователь может выйти из аккаунта.

### Желательно

* Хранить refresh token безопасно.
* Дать возможность отключить интеграции.
* Показывать статус подключения: Calendar подключен / Sheets подключен.

---

## 5.2. Главная страница “Сегодня”

Главная страница должна показывать:

1. Текущую дату и день недели.
2. Ближайшие уроки на сегодня.
3. Свободные временные окна.
4. Задачи на сегодня.
5. Ближайшие дедлайны.
6. Открытые долги.
7. Ближайшие тесты.
8. Мини-статистику:

   * сколько задач запланировано;
   * сколько выполнено;
   * сколько долгов открыто;
   * сколько тестов в ближайшие 7 дней.

### Пользователь должен иметь возможность:

* отметить задачу выполненной;
* перенести задачу на другой день;
* добавить быструю задачу;
* открыть детали урока, теста, долга или предмета;
* создать напоминание.

---

## 5.3. Страница “Неделя”

Страница должна показывать неделю в виде календарной сетки:

* дни недели;
* уроки из Google Calendar;
* задачи;
* тесты;
* дедлайны;
* свободные слоты.

### Функции

* Переключение между неделями.
* Добавление задач в конкретный день.
* Drag-and-drop задач по дням и временным слотам.
* Подсветка перегруженных дней.
* Отображение задач из Google Sheets.
* Возможность синхронизации с Google Calendar.

---

## 5.4. Интеграция с Google Calendar

### Приложение должно уметь:

* получать события календаря за выбранный период;
* определять учебные занятия;
* показывать занятия в расписании;
* определять свободные слоты между событиями;
* создавать новые события, если пользователь хочет запланировать задачу;
* обновлять созданные приложением события;
* не удалять чужие события без подтверждения.

### Правила обработки событий

Событие считается уроком, если:

* оно находится в учебном календаре;
* или содержит название предмета;
* или имеет специальный тег/цвет;
* или пользователь вручную отметил его как урок.

---

## 5.5. Интеграция с Google Sheets

### Цель

Подтягивать данные из таблицы “Планирование своей недели”.

### Возможные данные из таблицы

* задачи;
* цели недели;
* дедлайны;
* приоритеты;
* статусы;
* заметки;
* предметы.

### Требования

* Пользователь указывает ссылку на Google Sheets.
* Приложение читает выбранный лист.
* Пользователь может настроить соответствие колонок:

  * название задачи;
  * дата;
  * дедлайн;
  * предмет;
  * приоритет;
  * статус;
  * комментарий.
* Приложение импортирует строки как задачи.
* При повторной синхронизации приложение не должно создавать дубликаты.
* Изменения в Google Sheets должны обновляться в приложении.

### На первом этапе

Достаточно сделать ручную кнопку “Синхронизировать Sheets”.

---

## 5.6. Личный кабинет колледжа

Так как доступность API неизвестна, нужно предусмотреть 3 режима интеграции.

### Режим 1: Ручной ввод

Пользователь сам добавляет:

* оценки;
* долги;
* тесты;
* домашние задания;
* предметы.

Это обязательно для MVP.

### Режим 2: Импорт файла

Если личный кабинет позволяет выгрузить данные, приложение должно поддерживать импорт:

* CSV;
* XLSX;
* JSON;
* HTML-файла страницы.

### Режим 3: Автоматический импорт

Возможные варианты:

* официальный API личного кабинета;
* парсер страниц после авторизации;
* браузерное расширение, которое читает данные со страницы и отправляет в приложение.

Для MVP автоматический парсинг не обязателен.

* свободные слоты в Google Calendar;

* задачи;
* дедлайны;

## 6. Нефункциональные требования

## 6.1. Производительность

* Главная страница должна загружаться менее чем за 2 секунды при обычном использовании.
* Синхронизация Google Calendar должна выполняться за разумное время.
* Повторные данные должны кэшироваться.

## 6.2. Безопасность

* OAuth-токены должны храниться в зашифрованном виде.
* Пароли от личного кабинета колледжа не должны храниться в открытом виде.
* Лучше не хранить пароль от личного кабинета вообще, если можно использовать ручной импорт или расширение.
* Все запросы должны идти через HTTPS.
* Пользователь должен иметь возможность удалить все свои данные.

## 6.3. Надежность

* Если Google Calendar недоступен, приложение должно показать последние сохраненные данные.
* Если Google Sheets недоступен, приложение должно показать ошибку синхронизации, но не ломаться.
* При ошибке импорта пользователь должен видеть понятное сообщение.

## 6.4. Масштабируемость

На первом этапе приложение рассчитано на одного пользователя, но архитектуру лучше делать так, чтобы потом можно было добавить нескольких пользователей.

## 11. User Stories

### Главная страница

* Как пользователь, я хочу открыть приложение и сразу увидеть, что у меня сегодня по учебе.
* Как пользователь, я хочу видеть ближайшие дедлайны, чтобы ничего не забыть.
* Как пользователь, я хочу видеть свободные окна, чтобы запланировать задачи.

### Google Calendar

* Как пользователь, я хочу подключить Google Calendar, чтобы расписание подтягивалось автоматически.
* Как пользователь, я хочу видеть уроки из календаря в недельном плане.

### Google Sheets

* Как пользователь, я хочу подключить таблицу с недельным планированием.
* Как пользователь, я хочу, чтобы задачи из таблицы появлялись в приложении.

### Долги

* Как пользователь, я хочу видеть все долги в одном списке.
* Как пользователь, я хочу закрывать долг и видеть прогресс.
* Как пользователь, я хочу запланировать долг на свободное время.

### Оценки

* Как пользователь, я хочу добавлять оценки по предметам.
* Как пользователь, я хочу видеть средний балл по каждому предмету.

### Тесты

* Как пользователь, я хочу видеть ближайшие тесты.
* Как пользователь, я хочу получать напоминание перед тестом.

## 14. Бизнес-логика планирования

### Определение свободных слотов

1. Получить события из Google Calendar.
2. Отсортировать события по времени.
3. Задать рабочий диапазон дня, например 08:00–22:00.
4. Найти промежутки между событиями.
5. Исключить слишком короткие промежутки, например меньше 30 минут.
6. Показать свободные слоты пользователю.

### Приоритет задач

Срочность можно считать по формуле:

```text
score = priority_weight + deadline_weight + debt_weight + test_weight
```

Где:

* высокий приоритет дает больше очков;
* близкий дедлайн дает больше очков;
* долги получают дополнительный вес;
* подготовка к тестам получает дополнительный вес.

### Рекомендация задач на день

Приложение должно предлагать задачи, которые:

* имеют близкий дедлайн;
* относятся к долгам;
* связаны с ближайшими тестами;
* занимают подходящее количество времени для свободного слота;
* еще не выполнены.

## 18. Риски проекта

### Риск 1: Личный кабинет не имеет API

Решение:

* начать с ручного ввода;
* добавить импорт файла;
* позже сделать браузерное расширение.

### Риск 2: Google OAuth и scopes могут быть сложными

Решение:

* сначала подключить только Calendar read-only;
* потом добавить создание событий;
* потом добавить Sheets.

### Риск 3: Сложная синхронизация

Решение:

* хранить external_id у импортированных объектов;
* вести SyncLog;
* не создавать дубликаты;
* делать ручную синхронизацию на MVP.

### Риск 4: Проект может стать слишком большим

Решение:

* начать только с Dashboard + Calendar + ручные долги/тесты;
* Google Sheets подключить вторым этапом;
* личный кабинет оставить на поздний этап.

## 21. Итоговая формулировка проекта

**StudentHub** — это персональное веб-приложение для студента, которое объединяет расписание, недельное планирование, задачи, долги, оценки, тесты и напоминания в одном интерфейсе. Система помогает видеть учебную картину целиком, планировать свободное время, не забывать дедлайны и постепенно закрывать долги.

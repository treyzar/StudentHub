# Подключение Google Cloud для StudentHub

Эта инструкция объясняет, как получить **Client ID** и **Client Secret** в Google
Cloud Console, чтобы StudentHub мог входить в твой Google аккаунт и читать
Google Calendar и Google Sheets.

Все шаги расписаны максимально подробно — даже если ты никогда не работал с
Google Cloud, у тебя всё получится. Делай по порядку, ничего не пропуская.

---

## Что нужно перед началом

1. Аккаунт Google (тот же, чьи календарь и таблицы будешь использовать).
2. Браузер.
3. Около 10–15 минут времени.

Платить ничего не нужно: всё, что мы будем делать, бесплатно и не требует
привязки карты.

---

## Шаг 1. Создаём проект в Google Cloud

1. Открой страницу <https://console.cloud.google.com/>.
2. Войди под своим аккаунтом Google. Если попросят согласиться с условиями
   использования — соглашайся.
3. В шапке слева, рядом с надписью **Google Cloud**, нажми на выпадающий список
   проектов (там может быть написано «Select a project» или название старого
   проекта).
4. В открывшемся окне нажми кнопку **NEW PROJECT** (Новый проект) в правом
   верхнем углу.
5. Заполни форму:
   - **Project name** — например, `studenthub`.
   - **Organization / Location** — оставь как есть («No organization»).
6. Нажми **CREATE**.
7. Подожди 10–20 секунд, пока проект создаётся. Когда он будет готов, в шапке
   страницы появится надпись с его названием. Если нет — снова открой выпадающий
   список и выбери проект `studenthub` вручную.

---

## Шаг 2. Включаем нужные API

Нужно включить три API: Google Calendar, Google Sheets и Google Drive (он нужен,
чтобы видеть список твоих таблиц).

1. В левом верхнем углу нажми на «гамбургер» (☰), затем **APIs & Services →
   Library** (Библиотека).
2. В строке поиска введи **Google Calendar API** и нажми Enter.
3. Открой найденный сервис и нажми синюю кнопку **ENABLE** (Включить).
4. Когда API включится, нажми «← Back» или снова открой **Library**.
5. Повтори то же для **Google Sheets API**: найти → ENABLE.
6. Повтори то же для **Google Drive API**: найти → ENABLE.

Если кнопка показывает не «ENABLE», а «MANAGE» — значит API уже включён, ничего
делать не нужно.

---

## Шаг 3. Настраиваем экран согласия (OAuth consent screen)

Это окно, которое Google показывает пользователю, когда приложение просит
доступ к его данным.

1. Слева в меню выбери **APIs & Services → OAuth consent screen**.
2. Если откроется страница с предложением выбрать тип пользователей — выбери
   **External** и нажми **CREATE**.
   - «External» означает, что войти сможет любой Google аккаунт, в том числе
     твой личный. Для домашнего проекта это правильный выбор.
3. На экране **App information** заполни:
   - **App name** — `StudentHub` (или любое имя, оно будет видно тебе же при
     входе).
   - **User support email** — твой email.
   - **App logo** — можно пропустить.
4. На экране **Developer contact information** укажи свой email и нажми
   **SAVE AND CONTINUE**.
5. На экране **Scopes** ничего добавлять не нужно. Просто нажми
   **SAVE AND CONTINUE**.
6. На экране **Test users** нажми **+ ADD USERS** и добавь свой email
   (тот, чьим календарём и таблицами будешь пользоваться). Нажми
   **SAVE AND CONTINUE**.
   - Это важный шаг: пока приложение в режиме «Testing», входить смогут
     только те аккаунты, которые добавлены в этот список.
7. На итоговом экране нажми **BACK TO DASHBOARD**.

> Если в новом интерфейсе Google Cloud вместо «OAuth consent screen» показан
> раздел **Branding** и **Audience** — заполни там те же поля: имя приложения,
> email, и в **Audience** выбери **External**, после чего на вкладке
> **Audience → Test users** добавь свой email.

---

## Шаг 4. Создаём OAuth Client ID

1. Слева в меню выбери **APIs & Services → Credentials**.
2. Сверху нажми **+ CREATE CREDENTIALS → OAuth client ID**.
3. **Application type** — выбери **Web application**.
4. **Name** — например, `StudentHub Web Client`.
5. В блоке **Authorized JavaScript origins** нажми **+ ADD URI** и добавь:
   - `http://localhost:21995` (адрес фронтенда при локальном запуске)
   - `http://localhost:8080` (адрес бэкенда при локальном запуске)

   Если ты запускаешь приложение в Replit, добавь ещё свой публичный адрес,
   например `https://<id>-<slug>.kirk.replit.dev`.
6. В блоке **Authorized redirect URIs** нажми **+ ADD URI** и добавь точный
   адрес, по которому Google будет возвращать пользователя после входа:
   - `http://localhost:8080/api/google/callback` — для локального запуска.
   - Если используешь Replit:
     `https://<id>-<slug>.kirk.replit.dev/api/google/callback`.

   Адресов можно добавить несколько (например, и локальный, и Replit), Google
   разрешит вход с любого из них.
7. Нажми **CREATE**.
8. Откроется окно с **Client ID** и **Client secret**. Скопируй оба значения и
   сохрани их в надёжном месте (например, в менеджере паролей). Их же можно
   потом скачать как JSON файл кнопкой **DOWNLOAD JSON**.

> Если ты потерял Client secret, его можно посмотреть позже, открыв созданный
> client в **Credentials** и нажав «значок глаза».

---

## Шаг 5. Прописываем ключи в проекте

Открой корень проекта StudentHub и создай файл `.env` (если его ещё нет) с
таким содержимым:

```env
# === Google OAuth ===
GOOGLE_CLIENT_ID=...вставь сюда Client ID...
GOOGLE_CLIENT_SECRET=...вставь сюда Client secret...
# Адрес, на который Google вернёт пользователя после входа.
# Должен полностью совпадать с тем, что ты добавил в Authorized redirect URIs.
GOOGLE_REDIRECT_URI=http://localhost:8080/api/google/callback
```

Файл `.env` хранится только у тебя на компьютере и не попадает в Git
(он уже в `.gitignore`). Никогда не выкладывай Client secret в открытый
репозиторий.

Дальше переменные подхватятся при запуске бэкенда (см.
`docs/local-setup.ru.md`).

---

## Шаг 6. Проверяем работу

1. Запусти приложение локально (см. `docs/local-setup.ru.md`).
2. Открой <http://localhost:21995/settings>.
3. В блоке **Google аккаунт** нажми **Войти через Google**.
4. Откроется страница Google. Войди под тем же аккаунтом, который добавлял в
   Test users.
5. Google покажет предупреждение, что приложение «не проверено» — это
   нормально для проекта в статусе Testing. Нажми **Advanced → Go to
   StudentHub (unsafe)**.
6. Подтверди запрашиваемые разрешения (Calendar, Sheets, Drive readonly).
7. После согласия Google вернёт тебя на страницу настроек, появится твой email
   и аватар, а ниже — ближайшие события календаря и список последних таблиц.

---

## Возможные ошибки

| Сообщение | Что делать |
| --- | --- |
| `Error 400: redirect_uri_mismatch` | Проверь, что адрес в `GOOGLE_REDIRECT_URI` **дословно** совпадает с тем, что ты ввёл в **Authorized redirect URIs** (включая http/https, порт и `/api/google/callback`). |
| `Access blocked: This app's request is invalid` | Скорее всего ты не сохранил OAuth consent screen или не добавил себя в Test users. Вернись к шагу 3. |
| `Error 403: access_denied` | Аккаунт, которым ты входишь, не добавлен в Test users. Добавь его на вкладке **Audience → Test users**. |
| Кнопка «Войти через Google» в настройках просит задать переменные | Значит `.env` не попал в окружение. Перезапусти бэкенд после правки `.env`. |

---

## Что делать, когда захочешь убрать «unverified» предупреждение

Для домашнего использования это не нужно. Но если когда-нибудь захочешь
выложить приложение для других людей, нужно будет:

1. На странице **OAuth consent screen** нажать **PUBLISH APP** и заполнить
   обязательные поля (домашняя страница, политика конфиденциальности).
2. Пройти проверку Google. Это бесплатно, но может занять несколько недель.

Пока StudentHub — твой личный инструмент, оставайся в режиме **Testing** и
просто держи себя в списке Test users.
